## 자바스크립트 언어가 다른 언어와 다른 특징 3가지를 작성해주세요
우선 자바스크립트는 웹을 움직이는 대표적인 언어로, 사실상 웹 브라우저가 다른 언어는 이해 못해도 자바스크립트는 유일하게 이해하는 언어로서 자리잡고 있다. 
그리고 자바 스크립트는 JIT 컴파일 방식을 사용하여 실행 시점에 필요한 부분을 즉석에서 컴파일 하는 방식을 사용하기에 다른 언어에 비해 실행과정이 유연한 언어라고 소개할 수 있다.
마지막으로 자바스크립트는 HTML 안에 바로 연결하여 웹 페이지를 만들 수 있지만, 다른 언어는 이러한 과정을 거치기 위해선 다른 방식으로 연결해야 하기 때문에, 이러한 점은 자바스크립트 언어만의 특징이라 할 수 있다.

## 변수에 대해 설명하고 let, const, var 차이점을 작성해주세요
변수는 코딩에 있어서 가장 기본적인 개념으로, 원하는 값을 받아와 저장 및 출력하거나 반복 횟수를 세아리거나 복잡한 계산을 거칠 때 등 어떤 값 하나로 고정되있지 않은 수이다. 세 가지 변수 선언 유형 중 let은 중복 선언이 불가능하지만 재할당이 가능한 유형이고, const는 중복 선언도 불가능하고 재할당도 불가능한 상수 선언과 같다. var은 중복 선언과 재할당 모두 가능하며, let과 const가 블록 단위 스코프인 {} 안에서 선언 시 그 안에서만 참조 가능하다는 점과 달리 var은 같은 파일이라면 어디서든 참조가 가능하다는 차이가 있다.

## ==과, ===의 차이점을 작성해주세요
'=='은 값을 비교할 때 자료형을 자동으로 변환하여 두 값을 비교하고, '==='는 값을 비교할 때 자료형의 차이까지도 고려하여 값을 비교한다. 예를 들어 7과 "7"가 있다고 하자. 이때 '=='로 값을 비교하면 이 둘의 자료형이 다를 지 언정 true가 결과로 나온다. 하지만, '==='으로 비교하면 이들의 자료형은 7은 정수형, "7"는 문자형으로 자료형이 다르기 때문에 false가 결과로 나온다.

## var 변수 타입을 지양해야하는 이유를 말해주세요
규모가 큰 프로젝트에서 코드가 길어지다보면 많은 함수가 만들어지고, 여러 함수 내에서 변수가 만들어진다. 이때, var은 값을 재할당 할 수도 있고 중복선언도 할 수 있기에, 코드가 복잡해지면서 변수 관리가 어려워질 수 있다. 이는 예상하지 못한 동작이나 오류의 원인을 찾기 힘들어지는 현상으로 이어질 수 있다. 하여, var 사용은 지양하고 let과 const를 통해 필요한 경우에 따라 구분해 사용하는 것이 좋다.

## function foo() {}와 var foo = function() {} 사이에서 foo 사용의 차이에 대해 작성해주세요
fuction foo() {}에서 foo 같은 경우 함수를 바로 선언하는 방식으로, 만일 함수를 선언하기도 전에 함수를 호출 " foo(); "을 해도 잘 작동한다. 하지만, var foo = function() {} 같은 경우엔 함수 선언을 변수에 할당하는 것이다. 이때, 보통 변수는 선언하기 전에 사용하면 오류가 난다. 즉, 함수 선언을 변수에 할당하기 전에 함수를 호출 " foo(); "하고자 하면 함수가 아니라며 오류 메시지가 뜬다. 결과적으론 fuction foo() {}은 선언 전후 상관없이 함수를 호출할 수 있고, var foo = function() {}는 선언 후에만 함수를 호출할 수 있다.

## 화살표 => 함수 문법에 대한 사용 예시를 작성해주세요. 이 문법은 다른 함수와 어떻게 다른지 작성해주세요

화살표 함수는 다른 함수들과 달리 function이라는 키워드를 사용하지 않고도 더 간단히 함수를 표현할 수 있다. 그러한 이유로 map()이나 filter(), 콜백함수 등에서 함수를 간단히 표현하기 위해 사용한다.
예를 들어 'const a = arr.map(n => n*2);'는 map을 사용해 arr라는 배열의 각 요소에 2를 곱해 새로운 배열을 만들 때 사용된 화살표 함수의 예시이다.


## spread 문법에 대해 작성해주세요
spread 문법은 arr라는 배열이 있을 때, ...을 이용하여 {...arr}와 같은 형태의 문법으로 arr 속 배열을 객체로 나타내는 {} 기호로 감싸 표현해주는 문법이다. arr = ["a","b","c","d","e"]와 같은 배열이 있다면, spread 문법을 통하여 const obj = {...arr}로 obj라는 새 객체를 만들어 내, 
obj에는 {0:"a", 1:"b", 2:"c", 3:"d", 4:"e"} 와 같은 객체가 저장되는 것이다.

## 주제1 '배열관련 함수'
map은 배열의 각 값을 바꿔서 새로운 배열을 만드는 것이다.
const nums = [1,2,3,4,5];
const doubleNum = nums.map(num => num * 2);
console.log(doubleNum);
위 코드에서 'num => num * 2'는 배열의 각 요소를 num에 하나씩 받아 그 값들을 2배로 바꾸겠다는 의미로 해석할 수 있다.

filter는 조건에 맞는 값만 골라 새 배열을 만드는 것이다.
const users = [
 { name: "철수", age: 18 },
 { name: "맹구", age: 27 },
 { name: "유리", age: 24 }
];

const adults = users.filter(user => user.age >= 20);
console.log(adults);
위 코드에서 'user => user.age >= 20'는 배열의 각 요소를 user에 하나씩 받아 age의 값이 20 이상인 요소만 골라 새로운 배열을 만든다는 의미이다.

마지막으로 some은 배열 안에 조건을 만족하는 요소가 하나라도 있는지 확인하는 것이다.
const teenager = users.some(user => user.age < 20); 
이 코드는 배열 안에 age 값이 20보다 작은 요소가 하나라도 있으면 true를 반환한다는 의미이다.



## 주제2 '콜백함수'
콜백함수는 동시에 여러 일이 일어날 수 있는 웹 환경에서 한 함수에 전달되어 나중에 실행되는 함수이다. 예를 들어

function greet(name, callback) {
console.log("안녕하세요. " + name + "님!");
callback();	// 콜백함수 실행 

}

greet("홍길동", () => {

 console.log("오늘도 좋은 하루 되세요!!"); 

});

라는 코드가 있다면, 여기서 콜백함수는
() => {
  console.log("오늘도 좋은 하루 되세요!!");
}

이 부분으로 코드대로 해석하면, 위의 콜백함수가 greet이라는 함수에 전달되고, greet 함수 안에서 callback();이 실행될 때 호출된다. 결론적으로 전체 코드의 실행 순서는
"안녕하세요. 홍길동님!"이 먼저 출력되고, callback();이 실행되며, "오늘도 좋은 하루 되세요!!"가 나중에 출력된다.



## 주제3 '객체'
객체는 속성과 값을 한 쌍으로 묶어서
{key : value} 꼴로 저장하는 자료구조이다.
const user = {
	name: "철수",
 	age: "18",
 	isAdult: true
};
위 코드가 있을 때, "변수.key" 혹은 "변수['key']"와 같은 방법으로 value를 출력할 수 있다.
예시로 age를 출력하고 싶을 때, user.age 혹은 user['age']로 출력한다.

객체 관련 함수
Object.keys()는 속성(key)만 반환
Object.values()는 값(value)만 반환
Object.entries()는 속성과 값 둘다 반환
