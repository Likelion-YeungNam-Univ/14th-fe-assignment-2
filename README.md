# 멋쟁이사자처럼 2회차 과제

## 자바스크립트 언어가 다른 언어와 다른 특징 3가지
1. 실행 방식 (JIT 컴파일): 코드를 실행하는 시점에서 필요한 부분을 즉석에서 컴파일하여 실행한다.
2. 타입 시스템 (동적 타이핑): 변수가 특정 타입과 연결되지 않으며, 모든 타입의 값을 자유롭게 할당 및 재할당할 수 있다.
3. 메모리 관리 (자동): 가비지 컬렉션(Garbage Collection)을 통해 메모리를 자동으로 관리한다.
## 변수란? 
데이터를 저장하고 참조하기 위해 확보한 메모리 공간 또는 그 공간에 붙인 이름을 의미한다.
## let, const, var 차이점
| 구분 | var | let | const|
|---|---|---|---|
|재선언| 가능 | 불가능 | 불가능|
|재할당| 가능 | 가능 | 불가능 (상수)|

## ==과, ===의 차이점
* == (동등 비교): 피연산자의 자료형이 다를 경우 자동으로 타입 변환을 거친 후 값만 비교한다. (예: 1 == '1' 은 true)

* === (일치 비교): 타입 변환 없이 값과 자료형이 모두 일치해야 true를 반환한다. 실제로 코딩할 때는 자료형 변환이 일어나지 않는 === 를 주로 사용해야 한다.
## var 변수 타입을 지양해야하는 이유
1. 중복 선언 허용: 이미 선언된 변수 이름을 다시 사용해도 에러가 나지 않아 예기치 못한 값의 변경이 발생할 수 있다.
2. 범위 제어의 어려움: 선언 위치와 상관없이 어디서든 참조가 가능하여 코드의 복잡도를 높인다.

+)  JS의 버전 ES6 let 키워드가 나온 후 사용 권장하지 않는다.

## function foo() {}와 var foo = function() {} 사이에서 foo 사용의 차이
- function foo() {} (정규 표현): function 키워드를 사용하여 함수 이름과 코드를 직접 정의하는 기본 방식이다.
- var foo = function() {} (함수 표현법): 함수를 변수(foo)에 할당하여 사용하는 방식이다. (+화살표 함수 등을 활용해 더 간단히 표현하는 방법이 선호된다.)
## 화살표 => 함수 문법에 대한 사용 예시와 다른 함수 와의 차이점
- 예시> ```let sum = (a,b) => a + b; ```
- 특징:
    1. function 키워드 없이 더 간단하게 함수를 표현할 수 있다.

    2. 전달할 정보(매개변수)와 실행할 코드를 화살표로 연결하여 가독성을 높인다.
## spread 문법이란
- 정의: ... 기호를 배열 앞에 붙여서 사용하는 문법이다.

- 활용: 배열을 객체로 변환하거나 데이터를 전개할 때 사용한다. 예를 들어 const obj = {...arr};와 같이 작성하면 배열의 인덱스를 키(key)로, 요소 값을 값(value)으로 하는 객체를 생성할 수 있다.
# 주제 정리🦁

## 주제 1: 배열 고차 함수 (map, filter, some)
 자바스크립트 배열은 데이터를 효율적으로 가공하기 위한 내장 메서드를 제공한다. **map**은 배열의 모든 요소를 순회하며 새로운 형태의 배열을 생성하고, **filter**는 조건에 맞는 요소만 골라내어 추출한다. 또한 **some**을 활용하면 배열 내에 특정 조건을 만족하는 요소가 하나라도 있는지 간편하게 확인 가능하다. 이러한 메서드들은 코드의 가독성을 높이고 복잡한 로직을 간결하게 구현하도록 돕는다.

## 주제 2: 콜백 함수 (Callback Function)
콜백 함수란 다른 함수의 인자로 전달되어 특정 시점에 실행되는 함수를 의미한다. 자바스크립트에서는 함수를 값처럼 다룰 수 있는 특성(일급 객체) 덕분에 콜백 함수를 자유롭게 활용할 수 있다. 이는 주로 비동기 처리나 배열 메서드의 동작을 정의할 때 사용되며, 프로그램의 실행 순서를 제어하고 코드의 재사용성을 높이는 데 중요한 역할을 한다.

## 주제 3: 객체의 프로퍼티 (Property)
자바스크립트 객체는 키(Key)와 값(Value)의 쌍으로 구성된 프로퍼티들의 집합이다. 모든 데이터 타입이 프로퍼티의 값이 될 수 있으며, 특히 자바스크립트에서 배열은 내부적으로 인덱스를 키로 가지는 특수한 객체처럼 동작한다. 이러한 구조적 특성 덕분에 Object.assign()이나 Spread 문법을 통해 배열과 객체 간의 데이터 변환 및 복사가 유연하게 이루어진다.
