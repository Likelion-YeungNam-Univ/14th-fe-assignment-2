## 변수에 대해 설명하고 let, const, var 차이점을 작성해주세요
변수란 데이터를 저장하기 위한 공간이며, 값을 저장하고 재사용하기 위해 사용된다.
변수 선언 방식에는 var, let, const가 있다. 
var : 중복선언과 재할당이 가능하고 어디서든 참조 가능하다. 
let : 중복선언이 불가능하고 재할당이 가능하다. 블록 단위 스코프인 {} 안에서 선언 시 이 블록 안에서만 참조 가능하다.
const : 상수를 선언하고 중복선언과 재할당 모두 불가능하다. 블록 단위 스코프인 {} 안에서 선언 시 이 블록 안에서만 참조 가능하다.

## var 변수 타입을 지양해야하는 이유를 말해주세요
1. 중복선언이 가능하다.
같은 이름의 변수를 다시 선언해도 오류가 발생하지 않기 때문에, 실수로 같은 변수를 여러 번 선언해도 알아차리기 어렵다.
2. 함수 스코프를 가진다.
블록문 {} 안에서 선언하더라도 블록 밖에서 접근이 가능할 수 있다.
따라서 변수의 사용 범위를 예측하기 어렵고, 코드가 복잡해질수록 오류가 발생할 가능성이 커진다.

## function foo() {}와 var foo = function() {} 사이에서 foo 사용의 차이에 대해 작성해주세요
function foo() {} : 이 방식은 함수 선언문이다. 함수 선언문은 자바스크립트에서 호이스팅이 발생할 때 함수 전체가 위로 올라가기 때문에, 함수가 선언되기 전에도 호출할 수 있다. 따라서 코드의 어디에서든 비교적 자유롭게 사용할 수 있으며, 선언 이전에 호출해도 정상적으로 실행된다.
var foo = function() {} : 이 방식은 함수 표현식이다. 함수 표현식은 함수를 하나의 값으로 보고 변수에 할당하는 방식이다. 이 경우에는 변수 선언(var foo)만 호이스팅되고 실제 함수는 나중에 할당되기 때문에, 함수가 선언되기 전에 호출하면 오류가 발생한다. 따라서 반드시 함수가 정의된 이후에 사용해야 한다.

## 주제1 
- 콜백함수
콜백 함수는 함수의 인자로 전달되어, 특정 작업이 끝난 후 실행되는 함수이다. 주로 비동기 처리나 작업의 순서를 제어할 때 사용된다.
### 예시
function greet(name, callback) {
  console.log("안녕하세요 " + name + "님!");
  callback(); // 콜백 함수 실행
}

greet("홍길동", function() {
  console.log("오늘도 좋은 하루 되세요!");
});
위 코드에서 greet 함수는 이름을 출력한 후, 전달받은 callback 함수를 실행한다.
즉, callback 함수는 greet 함수의 작업이 끝난 뒤 실행되는 함수이다.

## 주제2 
- 배열과 객체의 차이
### 예시
// 배열
const scores = [90, 80, 70];

// 객체
const student = {
  name: "철수",
  score: 90
};
| 구분 | 배열 (Array) | 객체 (Object) |
|------|-------------|---------------|
| 데이터 저장 방식 | 순서(index)로 저장 | key(이름)로 저장 |
| 접근 방법 | 인덱스로 접근 → `arr[0]` | 키로 접근 → `obj.name` |
| 사용 목적 | 같은 종류의 데이터 목록 저장 | 서로 다른 속성 데이터를 묶을 때 |
| 구조 | `[ ]` 사용 | `{ }` 사용 |

## 주제3 
- 배열을 객체로 변환하는 방법
Object.assign은 여러 객체의 속성을 하나의 객체로 복사할 때 사용하는 메서드이다. 배열 역시 내부적으로 인덱스를 key로 가지는 객체 형태이기 때문에, 빈 객체에 배열을 복사하면 인덱스가 key, 요소 값이 value로 저장된 객체가 생성된다.
즉, 배열이 가지고 있는 인덱스 정보를 활용하여 객체 형태로 변환할 수 있다. 
### 예시
const colors = ['red', 'green', 'blue'];
const result = Object.assign({}, colors);
console.log(result); // {0: 'red', 1: 'green', 2: 'blue'}
이 방법은 배열의 구조를 그대로 유지하면서 객체로 변환할 수 있어 간단하게 사용할 수 있다.
