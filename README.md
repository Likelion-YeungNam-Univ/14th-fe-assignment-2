## 객체를 배열로 정리하는 방법 
Object.entries(객체)는
객체를 [key, value] 형태의 배열로 바꿔주는 메서드

const user = {
  name: "홍길동",
  age: 20,
  job: "개발자"
};

const result = object.entries(user);
console.log(result);

결과 
[
  ["name", "홍길동"],
  ["age", 20],
  ["job", "개발자"]
]
쓰는 이유
1. 반복문을 사용하기 쉽다.
2. map, filter 같은 배열 메서드를 사용가능하다
3. 다시 객체로 변환도 가능하다.

## 콜백함수 
콜백함수는 함수의 매개변수로 전달되어, 나중에 실행되는 함수이다.

function greet(name, callback) {
console.log("안녕하세요. " + name + "님!");
callback();	// 콜백함수 실행 
}

greet("홍길동", () => {
 console.log("오늘도 좋은 하루 되세요!!"); 
});

결과
안녕하세요. 홍길동님!
오늘도 좋은 하루 되세요!!

function greet(name, callback) {
  console.log("안녕하세요. " + name + "님!");
}

결과 - 안녕하세요. 홍길동님!
콜백 함수는 언제 실행할지를 결정하는 게 핵심이다.
그 실행 버튼이 callback()인것이다.

## ===을 더 많이 사용하는 이유
자동 형변환을 막아서 예측 가능한 비교를 하기 위해서
==은 값만 비교, ===은 자료형이랑 값을 같이 비교함.

"" == 0 // ture
"" === 0  // false

null == undefined // true
null === undefined // false

실무 예시

}

이유 : "false" -> true로 변환 이상한 비교 발생
